<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Deploy to Vercel and connect custom domain (Dennis)</name>
  <what-to-do>
    **Step 1: Push code to GitHub**
    ```bash
    git add .
    git commit -m "feat(04): polish and launch — SEO, analytics, privacy, sitemap"
    git push origin main
    ```

    **Step 2: Connect Vercel to your GitHub repo**
    1. Go to https://vercel.com → New Project
    2. Import your GitHub repo (notebooklm-mcp-cli or whatever the repo is named)
    3. Framework: Next.js (auto-detected)
    4. Root directory: leave as `.` (project root)
    5. Build command: `npm run build` (default — postbuild runs automatically)
    6. Click Deploy

    Wait for the first deploy to complete. Vercel gives you a `.vercel.app` URL to verify.

    **Step 3: Test on the Vercel preview URL**
    Visit `https://your-project.vercel.app`:
    - All 4 pages load
    - Contact form submits (Formspree env var is set from Task 1)
    - Favicon visible in browser tab
    - No broken images

    **Step 4: Connect custom domain**
    In Vercel: Project Settings → Domains → Add Domain: `www.FortPropertyDevelopments.com`

    Vercel will show you the DNS record to add. In your domain registrar (wherever you
    registered FortPropertyDevelopments.com):
    - Add CNAME record:
      - Name/Host: `www`
      - Value/Target: `cname.vercel-dns.com.` (trailing period required by some registrars)
    - Also add redirect: naked domain `FortPropertyDevelopments.com` → `www.FortPropertyDevelopments.com`

    DNS propagation: typically 5–60 minutes. Vercel auto-provisions SSL once DNS resolves.

    **Step 5: Verify live site**
    Once DNS propagates: visit https://www.FortPropertyDevelopments.com
    - HTTPS lock icon shows in browser
    - All 4 pages load
    - Submit contact form — confirm email arrives
  </what-to-do>
  <resume-signal>Type "deployed" once https://www.FortPropertyDevelopments.com is live over HTTPS and the contact form works on the live URL.</resume-signal>
