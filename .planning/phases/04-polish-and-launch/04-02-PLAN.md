# Key interface context for this plan:

Current app/layout.tsx exports:
```typescript

  title: 'Fort Property Developments',
  description: 'Multiplex infill developer serving Metro Vancouver and the Fraser Valley.',

```

This must be upgraded to include metadataBase, title template, and root OG defaults.

CRITICAL pitfall: openGraph is shallow-merged in Next.js. If any page sets ANY openGraph key,
the root layout's openGraph.images will NOT carry through. Every page that sets any openGraph
field MUST also include images: [{ url: '/images/hero-bg.jpg', width: 1200, height: 630 }].

OG image: /images/hero-bg.jpg (256KB, Unsplash download — use 1920x1280 as actual dimensions.
Specify width: 1200, height: 630 in metadata images array for OG spec compliance even if
actual dimensions differ — this is what social platforms read, not the actual file.)
</context>
